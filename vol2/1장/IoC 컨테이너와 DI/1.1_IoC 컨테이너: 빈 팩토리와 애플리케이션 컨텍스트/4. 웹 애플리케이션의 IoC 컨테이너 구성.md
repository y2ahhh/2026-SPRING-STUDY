## 웹 애플리케이션의 IoC 컨테이너 구성

`AnnotationContigServletWebServerApplicationContext`라는 **단 하나의 통합 컨테이너**만 띄어 `Controller`, `Service`, `Repository`를 한꺼번에 쉽고 빠르게 관리한다. 스프링6 기반의 신규 프로젝트는 직관적이고 단순한 단일 컨테이너 구조를 지향한다.

<br>

### 컨텍스트 계층 구조 핵심 원리
- **단방향 가시성(Visibility)**
  - 자식은 부모의 빈을 찾을 수 있지만 반대는 안된다.
- **빈 검색 순서**
  - 자식 컨텍스트에 도메인(`UserService`)이 없다면 부모 컨텍스트로 올라가서 찾고, 자식에게서 이미 존재한다면 부모에 동일한 이름의 빈이 있어도 자식의 빈이 우선권을 가진다.(Overriding)
- **논리적 분리**
  - 웹 계층<sup>SevletContext</sup>과 비즈니스/데이터 계층<sup>RootContext</sup>을 나누어 관리한다는 아키텍처적 목적 자체는 동일하다.

<br>

```
import jakarta.servlet.ServletContext;
import jakarta.servlet.ServletException;
import jakarta.servlet.ServletRegistration;
import org.springframework.web.WebApplicationInitializer;
import org.springframework.web.context.support.AnnotationConfigWebApplicationContext;
import org.springframework.web.servlet.DispatcherServlet;

// 순수 스프링6에서의 서블릿 애플리케이션 컨텍스트 등록 방법
public class MyWebInitializer implements WebApplicationInitializer {

    @Override
    public void onStartup(ServletContext servletContext) throws ServletException {
        // 1. 서블릿 애플리케이션 컨텍스트 생성 (자바 설정 클래스 기반)
        AnnotationConfigWebApplicationContext servletAppContext = new AnnotationConfigWebApplicationContext();
        servletAppContext.register(WebConfig.class); // @Controller, ViewResolver 등이 설정된 클래스

        // 2. 스프링의 핵심 서블릿인 DispatcherServlet 생성 및 컨텍스트 주입
        DispatcherServlet dispatcherServlet = new DispatcherServlet(servletAppContext);

        // 3. 톰캣(서블릿 컨테이너)에 DispatcherServlet 등록
        ServletRegistration.Dynamic registration = servletContext.addServlet("app", dispatcherServlet);
        registration.setLoadOnStartup(1);
        registration.addMapping("/"); // 모든 요청을 이 서블릿이 받도록 매핑
    }
}
```
`WebApplicationInitializer` 인터페이스를 구현해놓으면, 톰캣과 같은 `WAS`가 켜질 때 이 코드를 자동으로 실행하여 서블릿 컨텍스트로 등록해준다.

<br>

SpringBoot3 환경에서는 직접 적지 않고 Auto Configuration(자동 설정)엔진이 이 작업을 대신 해준다.
1. 애플리케이션이 구동되면 스프링 부트는 내부적으로 `AnnotationConfigServletWebServerApplicationContext`라는 통합 컨텍스트를 만든다.
2. 스프링 부트 내부에 내장된 `DispatcherServletAutoConfiguration`이라는 클래스가 활성화 된다.
3. 이 자동 설정 클래스가 DispatcherServlet을 생성하고, 애플리케이션 컨텍스트를 서블릿 컨텍스트로 자동으로 묶어서 내장 톰캣에 내포시킨다.
   ex) `@RestController`, `@Controller`
